# Unspark

A local sparkle remover for Google Gemini / Nano Banana images.

No cloud, no signup, no upload. Drop image → cleaned image lands in the same folder.

![Unspark idle state](https://raw.githubusercontent.com/L-tune/unspark/main/web/assets/spark.svg)

## Download

Grab the latest `Unspark.dmg` from [Releases](https://github.com/L-tune/unspark/releases).

Drag `Unspark.app` into `/Applications`. First launch: right-click → Open (Gatekeeper bypass for ad-hoc signed apps).

Requires macOS 11 (Big Sur) or later, Apple Silicon recommended.

## What it does

Google Gemini and the Nano Banana image API stamp a small ✦ watermark in the bottom-right corner of every generated image. Unspark detects the sparkle, masks it, and inpaints the gap using LaMa (large-mask inpainting from Samsung AI). Local. Offline. About 7 seconds per 2K image on Apple Silicon CPU.

## How it works

1. **Detection** — synthesizes a binary 4-pointed star template, matches it (multi-scale) against the top-hat transform of the bottom-right region. Top-hat suppresses global background brightness, so detection works on dark fabric, busy maps, and light wood floors equally well. Score range observed across Gemini samples: 0.55 to 0.80. Threshold for accepting: 0.40.

2. **Inpainting** — feeds the image and binary mask to LaMa via `simple-lama-inpainting`. LaMa uses Fast Fourier Convolutions, so its receptive field covers the whole image in one layer. Fills look seamless even at 100% crop on photographic textures.

3. **Interface** — vanilla HTML / CSS / JS designed in [Claude Design](https://claude.ai/design), wrapped in a 840×600 [PyWebView](https://pywebview.flowrl.com/) window. No Electron, no React, no CDN. Drag-drop file paths arrive via PyWebView's Cocoa hook into NSPasteboard, then process_file in Python writes the cleaned file next to the source.

## Build from source

```bash
git clone https://github.com/L-tune/unspark
cd unspark
./build.sh
open dist/Unspark.app
```

The build script:

- Downloads relocatable Python 3.11 from `python-build-standalone` (~20 MB)
- Installs torch + LaMa + opencv into bundled site-packages (~750 MB)
- Pre-downloads LaMa weights `big-lama.pt` (~200 MB)
- Renders `Unspark.icns` from `web/assets/spark.svg` via macOS `qlmanage`
- Ad-hoc signs the bundle

Final `Unspark.app` is ~1 GB. The `.dmg` compresses to ~530 MB.

Requires Homebrew Python 3.11 or 3.12 to bootstrap the build (`brew install python@3.11`). End users do not need any Python installed — everything runs from the bundle.

```

├── Info.plist                          # macOS bundle manifest

├── requirements.txt
├── Scripts/
│   ├── app.py                          # PyWebView entry + Bridge to JS
│   └── detect.py                       # sparkle detector (cv2 + multi-scale match)
└── web/
    ├── index.html                      # window UI (6 states)
    ├── styles.css                      # premium dark+light theme
    ├── app.js                          # vanilla state machine + bridge calls
    └── assets/
        └── spark.svg                   # logo + dissolution-particle source
```
